关于自动发货的详细用法

所谓自动发货，就是利用bot的仓库记录，自动选择正确的账号、人物，
到指定的游戏内扔掉指定的装备。要做到这一点要完成以下几个步骤：
=============================================================================
1、安装最新版本的loxxol map到你的kolbot中，其中会覆盖
libs\MuleLogger.js文件，这样以后bot记录装备时每件装备信息的
最后一行会自动添加账号信息。首次使用这个自动发货系统前，你需要
利用D2BotMuleLog把你的仓库遍历一边，使每件装备增加账号信息行，
以后就不用了。

2、建立仓库密码查询文件，使bot能够查到你的仓库账号的密码，
具体是在\d2bs\kolbot下建立一个名为acclist.txt的文本文件，
将仓库账号和密码用竖线分隔，每行天写一条，但为了节约工作量，
账号可以使用类似DOS中的通配符，*可以代替任意多个字符，?可以
代替1个字符。请看示例：
cangku01|123456
cangku02|123456
cangku03|123456
使用通配符的话这三行可以直接写成
cangku*|123456
这样的话就算你的仓库很复杂，一共也用不了几行就ok了。以后注意
保管好这个密码文件，不要不小心发给了别人哦。这个文件只需要编辑一次，长期有效。

3、在kolbot主界面添加一个Profile（最快的办法是复制一个现有的map线程），
入口文件仍选择d2botmap.dbj，profile名字设置成autotake,game.exe要选对，
其他地方随便填，甚至国度都不用管，bot会自动判断，填完保存，不要运行。

4、接下来就可以从你的仓库中选装备了，具体方法是：打开
tools\autotake\takelist.txt，这个文件开始时应该是空的，以后每次自动
发货都需要先清空这个文件，然后到bot的仓库管理界面（char viewer），
利用查找的办法找到你想拿取的装备，右键点击它，选择Copy Dscription，、
会把装备的详细描述复制到剪贴板中，然后粘贴到takelist.txt里边，你可以
一次选择多件装备依次粘贴，最后保存，文件可以不关。

5、随便启动一个map线程，选择你要拿装备的人物，建立一个普通难度带密码的游戏，
在罗格营地箱子处站好，输入指令-take，剩下的事情就基本不用管了，autotake线程
会自动启动，依次将装备扔到这个游戏里，建房人物背包有空间就会自动捡起扔下的
装备。如果你不想自动捡（比如交易给其他人的情况），只要离开箱子一段距离就可以了。
扔完装备后自动发货进程会自动关闭。

6、有几个可选变量可以设置，在D2BotMap.dbj的最开头部分，如下：

var autogametime=120000;//自动建游戏和加入游戏的尝试时间，一般采用默认值即可
var autotakeprofile="autotake";//执行自动发货的线程名，一般采用默认值即可
var mintakegametime=30000;//自动发货账号在游戏内时间至少30000豪秒（也就是30秒），防止封ip。
var checkitempick=true;//自动发货时，是否检测装备被捡起才离开游戏，打开这个，只要地上有装备人物
就不会离开游戏，你可以利用这一点顺便把想放进仓库的装备交给仓库号，但期间要主意保持地上至少有一件装备，
哪怕一个卷轴都可以，一旦地上没有装备了，仓库号就会开始记录装备并退出。

7、补充几个注意事项，第一是如果进程发生了中断，你可以在其他时间重新建一样
的房间名和密码，手动开启autotake线程即可。第二是autotake这个线程每次都会自动更换cdkey，但你不用设置它。
第三是，如果想中止进程，直接关掉autotake线程，然后将autotakegameinfo.txt文件清空即可。但这样做以后，
你需要再次执行-take来开启下一次自动发货。

8、以后每次使用自动发货，只需要再次执行第4、5两步就可以。

9、目前仍有些小bug，有时候bot会在界面外弹出一个错误提示，点确定就可以了，不会影响进程，不点的后果就是
autotake进程会卡在游戏里不动，但无其他影响。

10、如果仓库太多，不想遍历，也可以在没有账号信息行的装备后边参考已有信息手动填入账号信息行，也能达到效果。
